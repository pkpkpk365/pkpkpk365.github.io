# json 分类目录说明

## 本目录用途
- 存放 json 分类的代码或文本文件
- 使用本目录下的 `index.json` 登记索引
- 页面会读取索引后显示记录，并在需要时加载正文

## index.json 书写规则
每条记录固定使用以下字段：

```json
{
  "title": "[语言-用途] 标题",
  "desc": "简要说明",
  "file": "文件名"
}
```

### 字段说明
- `title`：页面展示标题，建议带用途关键词
- `desc`：页面展示说明，也参与搜索
- `file`：本目录中的实际文件名

## index.json 示例

```json
[
  {
    "title": "[JSON-配置] 示例配置文件",
    "desc": "演示 JSON 配置文件的基本结构",
    "file": "sample_config.json"
  }
]
```

## 新增内容的实际操作
1. 把代码文件放进本目录
2. 打开本目录下的 `index.json`
3. 追加一条记录
4. 提交到 GitHub 仓库

## 注意事项
- 不要把代码正文写进 json
- `file` 必须和真实文件名一致
- 标题尽量写清用途，否则后面很难检索
